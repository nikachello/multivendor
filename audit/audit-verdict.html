A real product with solid bones — but ship-blocking security defects for a live payments app.

The checkout path is genuinely well-built and the metadata basics are there. Yet today anyone can register as a platform admin, several server actions leak or mutate other merchants' data, two stored-XSS vectors ship on public storefronts, and subscription billing is silently broken. Fix the auth model and the shared tenant-check pattern first.

Critical verified Privilege escalation

Anyone can register as a platform admin (mass assignment on sign-up)

src/lib/auth/index.ts:49 — role: {"{ input: true }"}

The better-auth role field is declared user-writable, so POST /api/auth/sign-up/email with {"{ role: \"admin\" }"} creates a platform admin. assertAdmin trusts session.user.role verbatim → full access to every shop, user and order.

Fix → set input:false; assign roles only via an admin-gated action; add a create hook forcing non-privileged roles; scan the User table for rogue admins now.

saveTheme — no ownership check at all

src/lib/actions/theme.ts:15

Public "use server" RPC; pass any shopId to overwrite that shop's colors/fonts → storefront defacement.

Fix → assertOwnsShop(shopId) before update.

updateOrderStatus — trusts client shopId

src/lib/actions/order.ts:204

Only guard is existing.shopId !== shopId — two client values. Flip any order's status, restore its stock, email its customer.

Highverified

updateVariants — checks only updates[0]

src/lib/actions/variants.ts:122

Authorizes the first array element, then writes every id unscoped → overwrite any merchant's price/stock/SKU.

Fix → scope each write with product: {"{ shopId }"}.

getCouponsByShop — cross-tenant read

src/lib/actions/coupons.ts:9

No auth on the read path; enumerate any merchant's codes, discounts, usage and expiry.

Stored XSS via unvalidated analytics IDs

components/storefront/tracking/StorefrontGA4.tsx:24

Merchant ga4/pixel/ads IDs flow unvalidated into inline gtag() inside dangerouslySetInnerHTML → arbitrary JS on every shopper.

Fix → regex-validate IDs in updateShop; don't interpolate into script.

Highverified

Stored XSS via product JSON-LD

…/product/[productSlug]/page.tsx:109

JSON.stringify(jsonLd) doesn't escape </script>; a crafted product name breaks out of the ld+json block.

Fix → escape < > & to unicode; enforce productSchema.

Highverified

Unauthenticated file uploads

src/lib/uploadthing.ts:7

All three routes have empty middleware — anyone can upload to the platform's uploadthing account; no tenant scoping.

Fix → require session + assertOwnsShop in each middleware.

Highverified

13 of 14 action files skip schema validation

src/lib/actions/*.ts

Only order.ts uses zod; existing validators are never imported server-side. Root enabler of the XSS + data-integrity bugs.

Fix → safeParse a zod schema at the top of every action.

Payments (Bank of Georgia)
Highverified

Billing is silently broken — no one can go Pro

src/lib/bog.ts:95 — createVerify("SHA256withRSA")

That's a Java algorithm name; Node throws Invalid digest (reproduced), swallowed by catch{"{ return false }"} → every callback rejected → paid merchants never activate.

Fix → use "RSA-SHA256"; log config errors as 500 not 401.

Highverified

Callback has no idempotency or amount check

src/app/api/bog-callback/route.ts:42

Sets now + 30d on any signed sub_ callback — no dedup, no amount/currency check. Once billing is fixed, one captured callback = permanent free Pro.

Fix → store BOG payment id + reject repeats; verify amount; extend additively.

Platform hardening & discoverability
Highverified live

No HTTP security headers

next.config.ts — no headers()

Live check: no CSP, HSTS, X-Frame-Options (clickjacking on checkout/dashboard), nosniff, or Referrer-Policy on any host.

Fix → add an async headers() block.

Highverified live

No canonical → duplicate content on 3 hosts

storefront generateMetadata (all pages)

Same store on apex-path, subdomain and custom domain with zero canonical; og:url + sitemap point to the wrong (apex) host. Suppresses every merchant's rankings.

Fix → absolute alternates.canonical via getStorefrontUrl.

Highverified

Storefront over-fetch + unoptimized images

themes/*/FeaturedProductSection.tsx · next.config.ts:5

Whole category loaded to show 1–4 products (no React.cache; shop fetched 3×); 55/60 images ship unoptimized (~1.2MB, ~6.6s load).

Fix → slim take:N queries + cache; whitelist *.ufs.sh, drop unoptimized.
